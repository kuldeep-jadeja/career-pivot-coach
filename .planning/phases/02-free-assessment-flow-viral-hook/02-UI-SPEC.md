## Spacing Scale

Declared values (8-point scale from existing Tailwind config):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, form field internal padding |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding, card internal spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, page header spacing |
| 3xl | 64px | Page-level spacing, hero sections |

**Exceptions:**
- **Touch targets (mobile):** Minimum 44px height for buttons and form inputs (iOS/Android accessibility guideline)
- **Form field spacing:** 16px vertical gap between fields (comfortable mobile typing)
- **Multi-step progress indicator:** 32px between steps

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | zinc-50 (light) / zinc-950 (dark) | Background, page surfaces |
| Secondary (30%) | zinc-100 (light) / zinc-900 (dark) | Cards, form containers, nav |
| Accent (10%) | zinc-900 (light) / zinc-50 (dark) | **Reserved below** |
| Destructive | red-600 | Destructive actions only (none in Phase 2) |

**Accent reserved for:**
- Primary CTA buttons ("Calculate My AI Risk", "Download Share Card")
- Progress indicator fill (current step in multi-step form)
- Interactive states (hover, focus on primary actions)

**Risk Score Colors (semantic):**
- **Low Risk (0-20):** green-600 / green-400 (dark)
- **Moderate (21-40):** yellow-600 / yellow-400 (dark)
- **Elevated (41-60):** orange-600 / orange-400 (dark)
- **High (61-80):** red-600 / red-400 (dark)
- **Very High (81-100):** red-700 / red-500 (dark)

**Chart Colors (Recharts):**
- Layer 1 (AI Exposure): blue-500
- Layer 2 (Task Automation): purple-500
- Layer 3 (Industry): orange-500
- Layer 4 (Experience): green-500

## Layout

### Multi-Step Form Flow
- **Container:** max-width 640px, centered with 16px horizontal padding (mobile)
- **Step indicator:** Horizontal progress bar at top, 4px height, zinc-200 background, accent fill
- **Form sections:** Each step in separate card, 24px internal padding
- **Navigation:** "Back" and "Next" buttons, 16px gap, sticky bottom on mobile (300px padding-bottom for keyboard)

### Results Page
- **Hero section:** Risk score gauge (200px diameter), centered, 48px vertical padding
- **Breakdown sections:** Stacked cards with 16px gap
- **Layer visualization:** Horizontal bar chart, 100% width, 32px height per bar
- **Task breakdown:** Scrollable list, 8px gap between items, 44px min-height per touch target

### Share Card (PNG Export)
- **Dimensions:** 1200x630px (LinkedIn/Twitter/Facebook optimal)
- **Content:** Risk score (large), job title, branding, minimal text
- **Variants:** 3 framing options (proactive, curious, data-driven) - user selects before download

### Mobile Breakpoints
- **Mobile:** < 640px (single column, touch-optimized)
- **Tablet:** 640px - 1024px (forms remain centered, results expand slightly)
- **Desktop:** > 1024px (side-by-side layouts for comparison views)

## Interactivity

### Form States
- **Default:** zinc-200 border, zinc-50 background
- **Hover:** zinc-300 border transition (150ms)
- **Focus:** ring-2 ring-zinc-900 ring-offset-2, no outline
- **Error:** red-600 border, red-50 background, red-600 text below
- **Disabled:** zinc-100 background, zinc-400 text, cursor-not-allowed

### Transitions
- **Page navigation:** Fade in/out 200ms, no slide animations (accessibility)
- **Form step change:** Fade 150ms
- **Button states:** Background 150ms, border 150ms
- **Chart rendering:** Progressive draw 500ms (Recharts default)

### Loading States
- **Assessment calculation:** Spinner with "Analyzing your role..." text, centered
- **Chart loading:** Skeleton bars with zinc-200 background, pulse animation
- **Email sending:** Button disabled state with "Sending..." text

### Error Handling
- **Form validation:** Inline errors below each field, red-600 text, 14px
- **API failures:** Toast notification (use shadcn/ui toast), 5s duration, red background
- **Retry action:** "Try Again" button in error state

## Component Specifications

### Job Title Search (Combobox)
- **Component:** `<Command>` from shadcn/ui (Radix UI Combobox)
- **Behavior:** Client-side fuzzy search, debounced 300ms
- **Results:** Show top 5 matches with confidence scores (percentage badge)
- **No results:** Empty state with "No matches found" message
- **Mobile:** Full-width, 44px min-height, 16px font size

### Risk Score Gauge
- **Component:** Recharts `<RadialBarChart>`
- **Size:** 200px diameter (mobile/tablet), 280px (desktop)
- **Colors:** Semantic based on score range (see Color section)
- **Center text:** Score number (56px), "AI Risk" label (14px)
- **Animation:** Progressive fill 500ms on mount

### Layer Breakdown Chart
- **Component:** Recharts `<BarChart>` horizontal orientation
- **Height:** 32px per bar, 8px gap
- **Labels:** Left-aligned layer name, right-aligned percentage
- **Colors:** See Chart Colors in Color section
- **Tooltip:** Hover shows detailed breakdown

### Progress Indicator
- **Component:** `<Progress>` from shadcn/ui
- **Position:** Sticky top on mobile, fixed in form container on desktop
- **Steps:** Numbered dots (1, 2, 3), connected by line
- **Current step:** Accent color fill, 12px diameter
- **Completed:** Accent border, check icon
- **Upcoming:** zinc-300 border, empty

### Share Card Variants
- **Framing options:** 3 buttons (radio group) before download
  - "Proactive" — "I'm preparing for AI changes in my industry"
  - "Curious" — "Exploring how AI might impact my career"
  - "Data-Driven" — "Analyzing workforce trends with AI risk data"
- **Preview:** Show selected variant before download (100% width, 315px height, scale 0.5)
- **Export:** html2canvas at 2x scale for crisp images

## Responsive Behavior

### Mobile (< 640px)
- Single-column layout for all content
- Sticky form navigation (bottom: 0, 300px padding-bottom for keyboard)
- Full-width buttons (100% width, 44px min-height)
- Font size minimum 16px on inputs (prevent iOS auto-zoom)
- Touch targets minimum 44px (iOS accessibility)
- Charts scale to container width with min-height 200px

### Tablet (640px - 1024px)
- Forms remain max-width 640px, centered
- Results sections expand to 768px max-width
- Side-by-side layout for chart + legend
- Increased spacing (24px → 32px between sections)

### Desktop (> 1024px)
- Forms max-width 640px, centered with decorative background
- Results page 2-column layout (score left, breakdowns right)
- Hover states on all interactive elements
- Larger chart sizes (280px gauge diameter)

## Accessibility

- **Color contrast:** All text meets WCAG AA (4.5:1 minimum)
- **Focus indicators:** Visible ring on all interactive elements
- **Keyboard navigation:** Tab order follows visual flow, no keyboard traps
- **Screen readers:** Semantic HTML, ARIA labels on custom components
- **Touch targets:** Minimum 44x44px (iOS/Android guidelines)
- **Font scaling:** Supports user font size preferences (relative units)
- **Motion:** Respects prefers-reduced-motion (disable animations)

## Implementation Notes

### Session Storage Schema
```typescript
{
  "step": 1 | 2 | 3,
  "jobTitle": string | null,
  "selectedOccupation": { soc_code, title, confidence } | null,
  "industry": string | null,
  "yearsExperience": number | null,
  "completedAt": ISO timestamp | null
}
```

### Hydration Safety
- Wrap sessionStorage reads in `useEffect` with `isHydrated` flag
- Render skeleton/empty state until hydrated
- Prevents React hydration mismatch errors

### Email Template Styling
- Inline CSS only (Gmail compatibility)
- Max width 600px
- Single-column layout
- Semantic color values (not Tailwind classes)
- Test in Litmus/Email on Acid

### OG Image Generation
- Use `@vercel/og` for dynamic OG images
- Edge runtime for fast generation
- 1200x630px (LinkedIn/Twitter/Facebook optimal)
- Version query param for cache busting
- 1-hour cache headers
